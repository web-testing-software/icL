\section{Модификаторы управления}

\label{sec-modifiers}

Модификаторы управления позволяют изменить ход выполнения некоторых конструкций языка.

\subsubsection{\mintinline{icl}{if:not}}

\mintinline{icl}{if:not (@var == true)} эквивалентно \mintinline{icl}{if (!(@var == true))}, в этом случае уменьшения количестве скобок упрощает чтения кода.

\subsubsection{\mintinline{icl}{for:alt} - альтернативный параметрический цикл}

В параметрическом цикле меняется порядок выполнения действий. \mintinline{icl}{for:alt} гарантирует выполнение блока команд в минимум один раз.

Порядок действий конструкций \mintinline{icl}{for}:
\begin{icEnum}
    \item инициализация;
	\item проверка условий;
	\item выполнения блока команд;
	\item выполнения команды перехода на следующую итерацию.
\end{icEnum}

Порядок действий конструкций \mintinline{icl}{for:alt}:
\begin{icEnum}
    \item инициализация;
	\item выполнения блока команд;
	\item выполнения команды перехода на следующую итерацию;
	\item проверка условий.
\end{icEnum}

\subsubsection{\mintinline{icl}{while:not} - условное повторение кода}

\mintinline{icl}{while:not} будет выполнить блок команд, пока условие остаётся ложной.

\subsubsection{\mintinline{icl}{do while:not} - цикл с постусловий}

\mintinline{icl}{do while:not} будет выполнить блок команд, пока условие остаётся ложной. Так же как и \mintinline{icl}{do while} гарантирует что тело цикла будет выполнена минимум 1 раз.

\subsubsection{\mintinline{icl}{for:reverse} - прохождение коллекций}

\mintinline{icl}{for:reverse} будет пройти коллекцию в обратном порядке.

\subsubsection{\mintinline{icl}{filter:reverse} - выборочное прохождение коллекций}

\mintinline{icl}{filter:reverse} будет пройти коллекцию в обратном порядке.

\subsubsection{\mintinline{icl}{range:reverse} - частичное прохождение коллекций}

\mintinline{icl}{filter:reverse} будет пройти интервал в обратном порядке.

\subsubsection{\mintinline{icl}{emiter:alive} - второе дыхание}

В списке технологий \textit{icL} присутствует технология \textit{icL ALive}. Она позволяет восстановить работоспособность командного процессора после сбоя, но если исключение захватывается через \mintinline{icl}{emiter slot}, то она будет обработано им. Если нужно чтобы исключение обработалась через \textit{icL ALive} когда это доступно, укажите явно через данный модификатор.

Список технологии \textit{icL} доступен по ссылке: \ferror{https://gitlab.com/lixcode/icL/tree/standard/technologies\#technologies}.

\subsubsection{\mintinline{icl}{listen:ignore} - режим не беспокоить}

Конструкция \mintinline{icl}{listen} является частью технологий \textit{icL-Sync}, \mintinline{icl}{listen:ignore} позволяет создать узел, который не будет начать слушать при создании, а будет ожидать вызов метода \mintinline{icl}{Stack.listen()}. Больше информации можете найти в главе \ref{sync}.
